Redis

Redis基础篇

redis快速入门

认识NoSQL

NoSQL类型

  • 键值对型 (Redis)

  • 文档型 (MongoDB)

  • 图类型 (HBase)

  • 列类型 (Neo4j)

SQL VS NoSQL

#1 结构化(Structured) 非结构化

#2 关联的(Relational) 非关系

#3 SQL查询 非SQL (命令,函数, Restful)

#4 ACID Base

#5 磁盘 内存

#6 垂直扩展 水平扩展

​

SQL NoSQL
数据结构固定 数据结构不固定
对数据一致性, 安全性要求较高 对数据一致性,安全性要求不高,对性能要求高

认识Redis

Redis (Remote Dictionary Server) 远程词典服务器 2009年诞生

特征

  • 键值型, value支持不同类型

  • 单线程, 每个命令具备原子性(Redis 6.0 网络请求改为多线程, 核心命令仍是单线程)

  • 低延迟, 速度快(基于内存, 基于IO多路复用, 良好的编码( C语言 ))

  • 支持数据持久化

  • 支持主从集群, 分片集群

  • 多语言客户端

redis数据结构

  • KEYS (生产环境不建议使用)
  • DEL
  • EXISTS
  • EXPIRE
  • TTL

String

value是字符串, 不过根据字符串格式不同可以分为三类

  • string
  • int
  • float

字符串类型最大空间不能超过512m

String常见命令

  • SET key value

  • GET key

  • MSET key value [key value …]

  • MGET key [key …]

  • INCR key

  • INCRBY key integer

  • INCRBYFLOAT key float

  • SETNX key value (等价于 set key value nx)

  • SETEX key seconds value (等价于 set key value expire key seconds)

Key的层次结构

Redis中没有表的概念, 如何区分不同类型的key? (用户id是1, 商品id是1)

比如

项目名:业务名:类型:id

user相关的key : zs:user:1

product相关的key : zs:product:1
